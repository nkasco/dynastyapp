# Soul

## The Feeling

This app should feel like a private dynasty command center built by someone who watches the waiver wire too closely and likes software that respects attention. It is not a SaaS brochure, not a startup cosplay dashboard, and not a toy box of unrelated widgets.

It should feel personal, fast, and opinionated. The user should open it and immediately understand where their league stands, which players matter, which rosters are fragile, and where the next trade angle might be hiding.

## Product Principles

- Build for a friend group, not an abstract market.
- Prefer useful density over empty spectacle.
- Make imported data feel curated, not dumped.
- Treat every chart as a decision aid.
- Make uncertainty visible.
- Keep the system read-only toward Sleeper.
- Favor explainable analysis over fake precision.
- Let the user move from question to answer quickly.

## Visual Thesis

Dense, calm, and sharp: a scouting report crossed with a league war room. Dark and light themes should both work, but neither should feel like a default template. Use restrained team-color accents, crisp typography, confident spacing, and visual hierarchy that rewards scanning.

Player cards should feel like something worth collecting, comparing, and arguing about. They should be compact but expressive: name, team, position, age, trend, status, and the one or two numbers that matter right now.

## Interaction Thesis

The interface should feel responsive and deliberate.

- Filters should update quickly and preserve context.
- Cards should reveal more detail without making the page jump.
- Trade-building should feel like assembling evidence, not filling out a form.
- Import status should be transparent enough to trust but not loud enough to dominate the app.
- Motion should clarify state changes, selection, and hierarchy.

## Content Voice

The app should speak plainly. It should sound like a smart league mate who did the homework.

Good:

- "Last refreshed at 1:03 AM."
- "Three unmapped players need review."
- "This roster is thin at running back after Week 9."
- "The value gap is mostly age and pick liquidity."

Avoid:

- "Unlock next-level insights."
- "Dominate your league with AI."
- "Experience the future of fantasy analytics."
- "Revolutionary dynasty intelligence."

## What This App Refuses To Become

- A generic grid of metric cards.
- A public SaaS product pretending every user has the same needs.
- A black-box trade calculator with a fake decimal score.
- A Sleeper clone.
- A data warehouse with a thin UI pasted on top.
- A UI where the chart is pretty but the decision is still unclear.

## Taste Notes

The app should have Peter-Steinberger energy: technical, pragmatic, quietly exacting, and allergic to hand-wavy product theater. It should care about performance, edge cases, state restoration, and the small moments where software either earns trust or loses it.

Sharp details matter:

- Empty states should say what is missing and how to fix it.
- Loading states should preserve layout.
- Import failures should be specific.
- Player identity should be consistent across sources.
- Trade analysis should show its reasoning.
- Tables should be keyboard-friendly and not punish repeated use.

## North Star

The best version of this app helps a league mate answer:

"What should I do next?"

Not someday. Not after exporting a CSV. Right now, while the trade window is still open.
